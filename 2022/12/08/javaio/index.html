Leitura e escrita de arquivos em Java

Published qui., dez. 8, 2022 by Vitor Lobo Ramos

Estimated reading time: 8 min

Dados armazenados em variáveis e arrays são temporários eles são perdidos quando programa termina. Para retenção persistente de longo prazo podemos usar o registro destes dados em arquivos. Felizmente, várias linguagens orientadas a objetos (incluindo Java) fornecem maneiras de gravá-los e lê-los de arquivos (conhecidas como serialização e desserialização de objetos). Para ilustrar isso, recriamos alguns de nossos programas de acesso sequencial que utilizaram arquivos de texto, dessa vez armazenando e recuperando objetos de arquivos binários. O Java fornece a API java.nio.file para ler e gravar arquivos. A classe InputStream é a superclasse de todas as classes que representam um fluxo de entrada de bytes. Para ler um arquivo de texto você pode usar o método Files readAllBytes. O uso desse método é demonstrado na listagem a seguir.

1import java.io.IOException;
2import java.nio.file.Files;
3import java.nio.file.Paths;
4
5// em algum lugar do seu código
6
7String content = Files.readString(Path.of("resources", "work", "input.xml"));

Para ler um arquivo de texto linha por linha em uma estrutura List do tipo String, você pode usar o método Files.readAllLines.

1import java.io.IOException;
2import java.nio.file.Files;
3import java.nio.file.Paths;
4
5// em algum lugar do seu código
6
7String content = Files.readString(Path.of("resources", "work", "input.xml"));
8List<String> lines = Files.readAllLines(Paths.get(fileName));

Files.readAllLines usa codificação de caracteres UTF-8. Ele também garante que o arquivo seja fechado após a leitura de todos os bytes ou caso ocorra uma exceção. O método Files.lines permite ler um arquivo linha por linha, oferecendo um stream. Este fluxo pode ser filtrado e mapeado. Files.lines não fecha o arquivo depois que seu conteúdo é lido, portanto, ele deve ser agrupado dentro de uma instrução try-with-resource. No exemplo a seguir, os espaços em branco desnecessários no final de cada linha são removidos e as linhas vazias são filtradas.

1//lê todas as linhas e remove os espaços em branco (trim)
2//filtra linhas vazias
3//e imprime o resultado para System.out
4
5Files.lines(new File("input.txt").toPath())
6    .map(s -> s.trim())
7    .filter(s -> !s.isEmpty())
8    .forEach(System.out::println);

O próximo exemplo demonstra como filtrar linhas com base em uma determinada expressão regular.

1Files.lines(new File("input.txt").toPath())
2    .map(s -> s.trim())
3    .filter(s -> !s.matches("seuRegex"))
4    .forEach(System.out::println);

O próximo exemplo extrai uma linha começando com “Bundle-Version:” de um arquivo chamado MANIFEST.MF localizado na pasta META-INF (lembrando que o arquivo MANIFEST.MF contém metadados sobre o conteúdo do JAR). Ele remove o prefixo e todos os espaços em branco iniciais e finais..

 1package br.com.javaio.first;
 2
 3import java.io.IOException;
 4import java.nio.file.Files;
 5import java.nio.file.Paths;
 6import java.util.Optional;
 7import java.util.stream.Stream;
 8
 9public class ReadMANIFESTFile {
10    public static void main(String[] args) throws IOException {
11        String versionString = readStreamOfLinesUsingFiles();
12        System.out.println(versionString);
13    }
14
15    private static String readStreamOfLinesUsingFiles() throws IOException {
16        Stream<String> lines = Files.lines(Paths.get("META-INF", "MANIFEST.MF"));
17        Optional<String> versionString = lines.filter(s -> s.contains("Bundle-Version:")).map(e-> e.substring(15).trim()).findFirst();
18
19        lines.close();
20        if (versionString.isPresent()) {
21            return versionString.get();
22        }
23        return "";
24    }
25}

Crie um novo projeto Java chamado br.com.java.files. Crie a seguinte classe FilesUtil.java.

 1package br.com.java.files;
 2
 3import java.io.IOException;
 4import java.nio.file.Files;
 5import java.nio.file.Paths;
 6import java.nio.file.StandardOpenOption;
 7import java.util.List;
 8
 9public class FilesUtil {
10    public static String readTextFile(String fileName) throws IOException {
11        String content = new String(Files.readAllBytes(Paths.get(fileName)));
12        return content;
13    }
14
15    public static List<String> readTextFileByLines(String fileName) throws IOException {
16        List<String> lines = Files.readAllLines(Paths.get(fileName));
17        return lines;
18    }
19
20    public static void writeToTextFile(String fileName, String content) throws IOException {
21        Files.write(Paths.get(fileName), content.getBytes(), StandardOpenOption.CREATE);
22    }
23}

Você pode ler recursos de seu projeto ou de seu arquivo jar por meio da cadeia de métodos .getClass().getResourceAsStream() de qualquer objeto. O Java não impõe nenhuma estrutura a um arquivo noções como registros não fazem parte da linguagem Java. Portan- to, você deve estruturar os arquivos para atender os requisitos dos seus aplicativos. No exemplo a seguir, veremos como impor uma estrutura de registro chaveado a um arquivo. O programa desta seção cria um arquivo de acesso sequencial simples que pode ser usado em um sistema de contas a receber para monitorar os valores devidos a uma empresa por seus clientes creditícios. Para cada cliente, o programa obtém do usuário um número de conta, o nome e o saldo do cliente (isto é, o valor que o cliente deve à empresa por bens e serviços recebidos).

Os dados de cada cliente constituem um “registro” para ele. Esse aplicativo utiliza o número de conta como a chave de registro do arquivo que será criado e mantido em na ordem dos números das contas. O programa assume que o usuário irá inserir os registros em ordem de número de conta. Em um sistema abrangente de contas a receber (baseado em arquivos de acesso sequencial), seria fornecido um recurso de classificação, de modo que o usuário pudesse inserir o registro em qualquer ordem. Os registros seriam, então, classificados e gravados no arquivo. A classe CreateTextFile usa um Formatter para gerar Strings formatadas utilizando as mesmas capacidades de formatação que as do método System.out.printf. Um objeto Formatter pode gerar saída para vários locais, como para uma janela de comando ou um arquivo, como fazemos neste exemplo.

 1// Gravando dados em um arquivo de texto sequencial com a classe Formatter.
 2
 3import java.io.FileNotFoundException;
 4import java.lang.SecurityException;
 5import java.util.Formatter;
 6import java.util.FormatterClosedException;
 7import java.util.NoSuchElementException;
 8import java.util.Scanner;
 9
10public class CreateTextFile {
11
12    // envia uma saída de texto para um arquivo
13    private static Formatter output; 
14
15    public static void main(String[] args) {
16        openFile();
17        addRecords();
18        closeFile();
19    }
20    
21    // abre o arquivo clients.txt
22    public static void openFile() {
23
24        try {
25            output = new Formatter("clients.txt"); // abre o arquivo
26        } catch (SecurityException securityException) {
27            System.err.println("Não conseguiu gerar o arquivo...Terminando...");
28            System.exit(1); // termina o programa
29        } catch (FileNotFoundException fileNotFoundException){
30            System.err.println("Erro ao abrir o arquivo...Terminando...");
31            System.exit(1); // termina o programa
32        }
33    }
34
35    // adiciona registros ao arquivo
36    public static void addRecords() {
37        Scanner input = new Scanner(System.in);
38        System.out.printf("%s%n%s%n? ",
39                "Digite o número da conta, nome, sobrenome e saldo: ",
40                "Digite algo para sinalizar o fim do arquivo: ");
41
42        // faz um loop até o indicador de fim de arquivo
43        while (input.hasNext()) {
44
45            try {
46                // gera saída do novo registro para o arquivo; supõe entrada válida
47                output.format("%d %s %s %.2f%n", input.nextInt(), input.next(), input.next(), input.nextDouble());
48            } catch (FormatterClosedException formatterClosedException) {
49                System.err.println("Não conseguiu gerar o arquivo...Terminando...");
50                break;
51            } catch (NoSuchElementException elementException) {
52                System.err.println("Entrada inválida. Tente novamente!");
53                input.nextLine(); // descarta entrada para o usuário tentar de novo
54            }
55
56            System.out.print("?");
57
58        } // fim do while
59    } // fim do método addRecords
60
61    // fecha o arquivo
62    public static void closeFile() {
63        if (output != null)
64            output.close();
65    }
66} // fim da classe CreateTextFile

O que ocorre aqui é que o código acima estrutura um arquivo.txt e se o arquivo não existir, ele será criado. Se um arquivo existente estiver aberto, seu conteúdo será truncado e todos os dados no arquivo serão descartados. Se nenhuma exceção ocorrer, o arquivo é aberto para gravação e o objeto Formatter resultante pode ser usado a fim de gravar dados no arquivo. Os dados são armazenados em arquivos de modo que possam ser recuperados para processamento quando necessário. A Se- ção 15.4.1 demonstrou como criar um arquivo de acesso sequencial. Agora que criamos um arquivo com dados registrados sequencialmente nele, vamos criar uma classe que irá ler e exibir cada um desses registros no arquivocriado acima:

 1// Esse programa lê um arquivo de texto e exibe cada registro.
 2import java.io.IOException;
 3import java.lang.IllegalStateException;
 4import java.nio.file.Files;
 5import java.nio.file.Path;
 6import java.nio.file.Paths;
 7import java.util.NoSuchElementException;
 8import java.util.Scanner;
 9public class ReadTextFile {
10    private static Scanner input;
11
12    public static void main(String[] args) {
13        openFile();
14        readRecords();
15        closeFile();
16    }
17
18    // abre o arquivo clients.txt
19    public static void openFile() {
20        
21        try {
22            input = new Scanner(Paths.get("clients.txt"));
23        } catch (IOException ioException) {
24            System.err.println("Erro ao tentar abrir arquivo.");
25            System.exit(1);
26        }
27    }
28    // lê o registro no arquivo
29    public static void readRecords() {
30        System.out.printf("%-10s%-12s%-12s%10s%n", "Conta","Nome", "Sobrenome", "Resultados");
31
32        try {
33            while (input.hasNext()) {
34                // exibe o conteúdo de registro
35                System.out.printf("%-10d%-12s%-12s%10.2f%n", input.nextInt(), input.next(), input.next(), input.nextDouble());
36            }
37        } catch (NoSuchElementException elementException) {
38            System.err.println("Arquivo formado incorretamente. Terminando.");
39        } catch (IllegalStateException stateException) {
40            System.err.println("Erro ao ler do arquivo. Terminando.");
41        }
42    } // fim do método readRecords
43    // fecha o arquivo e termina o aplicativo
44    public static void closeFile() {
45        if (input != null)
46            input.close();
47    }
48} // fim da classe ReadTextFile
